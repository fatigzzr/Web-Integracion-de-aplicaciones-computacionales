Tarea 4: XML

Catálogo de Música y Registro de Empleados

Instrucciones

Catálogo de Música:

Diseña un XML que represente un catálogo de música. Debe incluir información sobre álbumes, artistas, géneros y canciones. (20 canciones)

Requisitos:
  • Cada álbum debe tener un título, año de lanzamiento, género y una lista de canciones.
  • Cada canción debe tener un título, duración, y si es un sencillo (true/false).
  • Cada álbum debe estar asociado con un artista, que tiene un nombre y un país de origen.

Registro de Empleados:

Diseña un XML que almacene información sobre empleados en una empresa. (20 empleados)

Requisitos:

  • Cada empleado debe tener un nombre, apellido, posición, y departamento.
  • Debe incluir el salario, fecha de contratación y si es tiempo completo o parcial.
  • El departamento debe tener un nombre y una localización.

Reflexión

Igual que en la actividad que hicimos en clase, me di cuenta de que la Inteligencia Artificial ayuda grandemente a generar registros sintéticos de manera satisfactoria cuando le das toda la información necesaria. Aunque bien, tuve que especificar ciertos aspectos para que el resultado fuera útil, como que los datos tenían que estar conectados o que debían estructurarse como tablas diferentes. Dentro de la tabla principal debían aparecer otras etiquetas que funcionaran como tablas o entidades relacionadas con los álbumes, artistas, géneros y canciones, y que se pudieran conectar entre sí, tal como sucede en una base de datos en SQL. Este detalle fue muy importante, ya que si no se dan instrucciones claras, la IA puede generar datos correctos pero desorganizados o sin relación alguna entre ellos.

Por otro lado, con esta tarea pude observar que dentro de un mismo archivo XML se puede tener una etiqueta principal que represente el nombre de la base de datos, y dentro de ella incluir etiquetas que representen las entidades junto con sus atributos, además de considerar las relaciones que existen entre esas entidades. Esto me permitió darme cuenta de que, en teoría, es posible almacenar toda la información de una base de datos dentro de un archivo XML, aunque reconozco que no sería lo más recomendable por temas de eficiencia y escalabilidad. Sin embargo, el ejercicio me sirvió para comprender cómo funciona la estructura jerárquica del XML y cómo se pueden modelar datos de manera similar a un esquema de base de datos relacional.

Además, me dio a entender que se podría mandar toda la información de una base de datos gestionada en un sistema de gestión tradicional hacia un archivo XML para enviarlo o visualizarlo en una página web. Esto resultaría bastante útil para validar que los datos insertados o que actualmente se encuentran en la base de datos están correctos, sin necesidad de conectarse directamente al sistema gestor. También observé que esta alternativa sería muy práctica en los casos en los que no se tiene permiso para editar los datos directamente, pero sí se permite su consulta a través de un archivo estructurado como XML. De esta forma, el archivo se convierte en una buena opción para revisar la información cuando no se tiene acceso completo a la base de datos, asegurando al mismo tiempo la integridad de los datos.
