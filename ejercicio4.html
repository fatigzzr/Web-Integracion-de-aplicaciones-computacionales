Ejercicio 4: XML + CSS + XSL

Sistema de gestión de una librería

Objetivos

Utilizar XSL para el pre-procesamiento de XML

Instrucciones

Eres el responsable del sistema de gestión de una librería en línea que ofrece libros en diferentes formatos (digitales y físicos). Para facilitar el intercambio, almacenamiento y procesamiento de la información, se requiere diseñar un archivo XML bien formado que represente el catálogo de libros disponible.


Estructura del Elemento <book>

Utilizando la IA genera un documento llamado libros.xml con 20 registros.

  • ISBN del libro (atributo isbn en el elemento <book>)
  • Título del libro (<title>)
  • Autor o autores (<author>)
  • Año de publicación (<year>)
  • Género (<genre>)
  • Precio (<price>)
  • Disponibilidad en stock (<stock>)
  • Formato (<format>)

Para mejorar la presentación de estos datos y hacerlos accesibles en una página web, debes realizar las siguientes tareas:

Crea una copia de tu archivo XML (libros3.xml):

  • Incluye la directiva <?xml-stylesheet?> para vincular el archivo XSLT que se encargará de transformar los datos XML en un formato HTML para su visualización.
  • <?xml-stylesheet type="text/xsl" href="libros.xsl"?>

Crea un archivo XSLT (libros3.xsl):

  • El archivo XSLT se encargará de transformar el contenido del archivo XML en una página HTML bien estructurada.
  • Debes incluir la directiva <?xml-stylesheet?> dentro del XSLT para enlazar un archivo CSS que estilizará la presentación de los datos.
  • En el proceso de transformación, calcula y muestra la suma total de los precios de los libros. Además, aplica un descuento del 10% a cada libro y muestra el precio con descuento.
  • Estructura el HTML generado para que cada libro se muestre dentro de una caja con su correspondiente información como ISBN, título, autor, género, precio, y formato.

Crear un archivo CSS (libros3.css):

  • Este archivo CSS se encargará de estilizar la página HTML generada por el XSLT.
  • Define estilos generales para la presentación de la librería, asegurándote de que cada libro tenga una presentación clara y organizada, con elementos como el título, autor, y precio destacados.
  • Asegura que la suma total de los precios y los precios con descuento se destaquen visualmente.

Transformación XML a HTML (libros3.xsl)

XSLT
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/css" href="libros3.css"?>
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
    
    <xsl:template match="/">
        <html>
            <head>
                <title>Catálogo de Libros - Librería Digital</title>
                <link rel="stylesheet" type="text/css" href="libros3.css"/>
            </head>
            <body>
                <div class="library">
                    <h1>Catálogo de Libros - Librería Digital</h1>
                    
                    <div class="books">
                        <xsl:for-each select="library/book">
                            <div class="book">
                                <div class="book-info">
                                    <div class="isbn">ISBN: <xsl:value-of select="@isbn"/></div>
                                    <div class="title"><xsl:value-of select="title"/></div>
                                    <div class="author"><xsl:value-of select="author"/></div>
                                    <div class="year">Año: <xsl:value-of select="publication_year"/></div>
                                    <div class="genre">Género: <xsl:value-of select="genre"/></div>
                                    <div class="format">Formato: <xsl:value-of select="format"/></div>
                                    <div class="stock">Stock: <xsl:value-of select="stock"/></div>
                                    
                                    <div class="prices">
                                        <div class="original-price">Precio Original: $<xsl:value-of select="price"/></div>
                                        <div class="discounted-price">Precio con Descuento (10%): $<xsl:value-of select="format-number(price * 0.9, '0.00')"/></div>
                                    </div>
                                </div>
                            </div>
                        </xsl:for-each>
                    </div>
                    
                    <div class="summary">
                        <h2>Resumen de Precios</h2>
                        <div class="total-original">Total Precios Originales: $<xsl:value-of select="format-number(sum(library/book/price), '0.00')"/></div>
                        <div class="total-discounted">Total con Descuento (10%): $<xsl:value-of select="format-number(sum(library/book/price) * 0.9, '0.00')"/></div>
                        <div class="savings">Ahorro Total: $<xsl:value-of select="format-number(sum(library/book/price) * 0.1, '0.00')"/></div>
                        <div class="book-count">Total de Libros: <xsl:value-of select="count(library/book)"/></div>
                    </div>
                </div>
            </body>
        </html>
    </xsl:template>
    
</xsl:stylesheet>

Reflexión

Si bien en la clase vimos cómo utilizar el XSLT para realizar búsquedas de los libros, ahora con el ejercicio guiado de generar el precio dado el descuento de los libros tengo mucho más presente el hecho de que XSLT te ayuda a “transformar” los datos. Esta transformación no se limita únicamente a mostrar información distinta, sino que puede tomar diversas formas: búsqueda de información específica, filtrado de datos según una condición determinada, o incluso el cálculo de nuevos valores a partir de la información ya existente. El ejemplo del cálculo del precio con descuento me pareció muy ilustrativo, porque demuestra que no solo se pueden presentar datos estáticos, sino también procesarlos y modificarlos según la lógica que se establezca en la hoja de estilo.

Al trabajar con XML, CSS y XSL, me doy cuenta de que ya es posible tener una especie de visualización o frontend de una base de datos, sin necesidad de apoyarse directamente en HTML. Esto me hizo reflexionar que, aunque HTML sea lo más común en el desarrollo web, no es estrictamente necesario para mostrar datos de manera organizada y atractiva. En este sentido, también me ayudó a comprender mejor el proyecto final del curso, ya que al principio no entendía cómo podríamos visualizar el backend sin construir una aplicación web o móvil desde cero. Ahora me queda más claro que con estas herramientas de transformación y estilización, se puede obtener una representación visual bastante práctica de la información.

Considero que es esencial conocer esta información, porque su uso tiene un enfoque muy práctico y puede resultar útil al trabajar en proyectos tanto en empresas como de manera individual. Muchas veces no se necesita construir una aplicación completa para verificar si el backend funciona correctamente o si los datos que se están procesando son los esperados. Con XML y XSLT ya se puede crear una capa intermedia de verificación y visualización sin un esfuerzo de programación tan grande.

En cuanto al uso de la Inteligencia Artificial en este ejercicio, me pareció una experiencia más positiva en comparación con otros casos anteriores. Creo que esto se debe a que las tareas que le pedimos fueron más concretas, como generar registros sintéticos o estructurar datos, en lugar de pedirle que tomara decisiones complejas, como diseñar toda una arquitectura con un único prompt. De esta manera entendí mejor cuál puede ser el rol de los ingenieros de software en el futuro: en lugar de dedicar tiempo a tareas repetitivas y mecánicas, podremos usar las herramientas que tenemos a nuestro alcance para automatizar esas partes y concentrarnos en actividades más valiosas, como diseñar arquitecturas, modelar bases de datos o tomar decisiones estratégicas en el desarrollo de software.
