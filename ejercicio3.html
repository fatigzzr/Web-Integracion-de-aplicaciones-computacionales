Ejercicio 3: XML + CSS

Sistema de gestión de una librería

Objetivos

Comprender el uso de XML y CSS

Instrucciones

Eres el responsable del sistema de gestión de una librería en línea que ofrece libros en diferentes formatos (digitales y físicos). Para facilitar el intercambio, almacenamiento y procesamiento de la información, se requiere diseñar un archivo XML bien formado que represente el catálogo de libros disponible.

Estructura del Elemento <book>

El archivo XML debe incluir la siguiente información para cada libro:

  • ISBN del libro (atributo isbn en el elemento <book>)
  • Título del libro (<title>)
  • Autor o autores (<author>)
  • Año de publicación (<year>)
  • Género (<genre>)
  • Precio (<price>)
  • Disponibilidad en stock (<stock>)
  • Formato (<format>)

Estilizado con CSS

Como encargado de gestionar una librería en línea, necesitas mejorar la presentación visual de tu archivo XML que contiene la lista de libros. Para ello, deberás crear un archivo CSS llamado library.css y vincularlo a tu XML utilizando la directiva <?xml-stylesheet?>: <?xml-stylesheet type="text/css" href="library.css"?>

Requisitos del CSS

El CSS debe cumplir con los siguientes requisitos:

Estilizar el contenedor principal (library):

  • Debe ocupar el 100% del ancho disponible.
  • Debe tener un padding de 20px.
  • El fondo debe ser de color #f0f0f0.
  • Se debe utilizar la fuente Verdana, Geneva, Arial, Helvetica, sans-serif.

Estilizar la caja que contiene todos los libros (books):

  • Debe ocupar el 100% del ancho disponible.
  • El fondo debe ser de color #cccccc.
  • Debe tener un padding de 15px.
  • Debe tener un margen inferior de 20px.

Estilizar cada libro individual (book):

  • Debe ocupar el 100% del ancho disponible.
  • El fondo debe ser de color #e0e0e0.
  • Debe tener un borde de 1px sólido de color #999.
  • Debe tener un padding de 10px.
  • Debe tener un margen inferior de 10px.

Estilizar los elementos dentro de cada libro:

  • Título (title): Debe tener un tamaño de fuente de 1.2em, estar en negrita, y tener un margen inferior de 5px.
  • Autor (author): Debe estar en cursiva y tener un margen inferior de 5px.
  • Género (genre), Precio (price), Stock (stock), Formato (format): Deben tener un margen inferior de 3px y ser mostrados en bloques.

Colores específicos:

  • Género: color #666.
  • Precio: color #009900.
  • Stock: color #ff0000.
  • Formato: color #0000ff.

Estilos para XML (library.css)

CSS
/* Estilizar el contenedor principal (library) */
library {
    width: 100%;
    padding: 20px;
    background-color: #f0f0f0;
    font-family: Verdana, Geneva, Arial, Helvetica, sans-serif;
}

/* Estilizar la caja que contiene todos los libros (books) */
books {
    width: 100%;
    background-color: #cccccc;
    padding: 15px;
    margin-bottom: 20px;
}

/* Estilizar cada libro individual (book) */
book {
    width: 100%;
    background-color: #e0e0e0;
    border: 1px solid #999;
    padding: 10px;
    margin-bottom: 10px;
    display: block;
}

/* Estilizar los elementos dentro de cada libro */

/* Título (title) */
title {
    font-size: 1.2em;
    font-weight: bold;
    margin-bottom: 5px;
    display: block;
}

/* Autor (author) */
author {
    font-style: italic;
    margin-bottom: 5px;
    display: block;
}

/* Año de publicación (publication_year) */
publication_year {
    margin-bottom: 3px;
    display: block;
}

/* Género (genre) */
genre {
    margin-bottom: 3px;
    display: block;
    color: #666;
}

/* Precio (price) */
price {
    margin-bottom: 3px;
    display: block;
    color: #009900;
}

/* Stock (stock) */
stock {
    margin-bottom: 3px;
    display: block;
    color: #ff0000;
}

/* Formato (format) */
format {
    margin-bottom: 3px;
    display: block;
    color: #0000ff;
}

Reflexión

En la clase vimos el uso del XML, el cual es un lenguaje de marcado bastante interesante. Anteriormente, no había escuchado sobre este tipo de archivo, y me sorprendió que se vea tan similar al HTML. Esto tiene sentido porque, al igual que el HTML, es un lenguaje de marcado que utiliza etiquetas para estructurar información. Sin embargo, la diferencia principal es que, mientras el HTML está diseñado para mostrar contenido en un navegador, el XML se utiliza principalmente para guardar, organizar o enviar información entre diferentes sistemas.

Algo que me llamó la atención es que en XML yo puedo personalizar las etiquetas. A diferencia del HTML, en donde las etiquetas ya vienen definidas, en XML yo puedo crear mis propias etiquetas según lo que necesite. Me imaginé esto como si fueran entidades o tablas en una base de datos, ya que puedo definir una etiqueta general, como “librería”, y dentro colocar atributos o subelementos como “ISBN”, “autor” o “título”. Esta forma de organizar datos me parece bastante lógica y flexible, sobre todo porque puedo adaptar el archivo XML a diferentes contextos.

También me pareció interesante que el archivo XML se pueda visualizar directamente en el navegador. Al principio me sorprendió, pero tiene sentido porque en clase vimos que XML cumple un papel parecido al de JSON: ambos sirven para transmitir información estructurada. Si puedo abrir un archivo JSON en un sitio web, entonces también debería poder ver un archivo XML, y al comprobarlo noté que efectivamente así funciona.

Finalmente, otro aspecto que me pareció llamativo es que en XML se pueden aplicar directamente formatos de estilo CSS, de forma parecida a como ocurre con HTML. Esto refuerza la idea de que CSS funciona como un lenguaje complementario que da estilo a lenguajes de marcado. Aunque no estoy seguro si JSON permite algo similar, lo que sí me queda claro es que XML tiene más relación estructural con HTML, y de ahí que pueda aprovechar estas mismas posibilidades de estilización.

En conclusión, conocer XML me ayudó a entender mejor la lógica detrás de los lenguajes de marcado y cómo se conectan con conceptos que ya conocía, como HTML, CSS y bases de datos.
